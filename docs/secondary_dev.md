# 二次开发指南

## 推荐扩展顺序

1. 先补充环境配置和真实接口模板
2. 再补全页面对象 locator
3. 再补 Flow 层的业务编排
4. 最后补 AI 提示词和自愈策略细节

## 不要这样做

- 不要把 locator 直接散落在测试用例里
- 不要把接口请求写进页面对象里
- 不要把 AI 请求硬编码到每个页面方法中
- 不要跳过 Flow 层直接在 case 中拼步骤

## 推荐这样做

- 页面对象只关注页面能力
- 复杂控件统一沉淀到 `components/`
- 跨页面业务逻辑沉淀到 `flows/`
- 业务域规则沉淀到 `domains/`
